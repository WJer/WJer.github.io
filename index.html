bootstrap-响应式启示

启示一

使用百分比

像栅格系统那样，通过设置宽度为百分比来适应浏览器的屏幕。

缺点：如果某行的元素数为5，那百分比应设置为20%；如果一行的元素为9，那就得人工计算；如果数目再次改变，那么又得在css样式中修改这个数值。

虽然得计算，但确实是个很好的办法。

启示二

这个思路不需要去计算除法。

我们在写导航栏的时候用的都是ul li等元素，那么以此为例：

ul>li{
    display:table-cell;
    width:1%; 
}

为什么是1%？
至今没想出来。

这样一设置，导航栏各li元素的宽度就会随着屏幕的宽度变化而变化，不论你怎么改变，它们都会平均的分配宽度。

关于table-cell值：
这个属性不能和float、position:absolute同时使用，它会被这些属性破坏。
IE8+浏览器及其他浏览器都支持。

启示三

学会使用媒体查询，根据不同的浏览器宽度使用不同的样式。

@media(最小宽度或最大宽度){
    ......
}

栅格系统可以说是bootstrap的核心之一，响应式设计就是建立在这个基础之上。

实现原理

定义容器，调整内外边距，平分12份，媒体查询

这四个合起来就能实现栅格系统。
ps：12等分是默认的，我们可以重新编译LESS来修改这个数值。

一个简单的例子：

<div class="container">
    <div class="row">
        <div class="col-md-2">1</div>
        <div class="col-md-2">1</div>
        <div class="col-md-2">1</div>
        <div class="col-md-2">1</div>
        <div class="col-md-2">1</div>
        <div class="col-md-2">1</div>
    </div>
</div>

响应式栅格

超小型屏幕：

  • <768px
  • 任何分辨率下都水平排列
  • 最大容器宽度时自动
  • .col-xs-
  • 可嵌套，可排序，可偏移

小型屏幕设备：

  • <992px &&="">=768px
  • 小于768时，垂直排列；超过768，水平排列
  • 最大容器宽度是750px
  • .col-sm-
  • 可嵌套，可排序，可偏移

中等屏幕设备：

  • <1200px &&="">=992px
  • 小于992，垂直排列；超过992，水平排列
  • 最大容器宽度是970px
  • .col-md-
  • 可嵌套，可排序，可偏移

大屏幕设备：

  • >1200px
  • 小于1200时，垂直排列；超过1200，水平排列
  • 最大容器宽度是1170px
  • .col-lg-
  • 可嵌套，可排序，可偏移

ps：向大兼容，小于媒体查询中的最小宽度，就不起作用了。

跨设备组合：由于向大兼容的原因，小就无法兼容，那么可以再设置一套样式，如下：

<div class="container">
    <div class="row">
        <div class="col-sm-6 col-md-2">1</div>
        <div class="col-sm-6 col-md-2">1</div>
        <div class="col-sm-3 col-md-2">1</div>
        <div class="col-sm-3 col-md-2">1</div>
        <div class="col-sm-3 col-md-2">1</div>
        <div class="col-sm-3 col-md-2">1</div>
    </div>
</div>

js-DOM节点类型-Element类型

Element类型

特征
nodeType:1;
nodeName:元素的标签名;
nodeValue: null;
parentNode: Document/Element;
ownerDocument: null;

特性
特性就是元素标签的属性，只是换了个名字而已。本人分析是为了区别元素标签的属性和DOM对象的属性而这样干的。

取得特性
两种方式：getAttribute()和直接访问DOM对象属性。

ps：并不是所有的特性都有对应的属性。对于一些公认的特性，才有与之对应的DOM对象的属性。

ps：在公认的特性中，有两类特殊的特性：style和onclick。两种方式取得的值不想同。由于这样的存在，开发人员在取得自定义特性使用getAttribute(),最普遍使用的还是DOM对象属性。

ps：ie浏览器中自定义特性也有对应的属性

设置特性
两种方式：setAttribute()和直接设置DOM属性。

ps：setAttribute()无法设置class和style特性。

ps：setAttribute()可以设置自定义特性，直接设置DOM属性只能设置属性，如果不是公共特性，并不能反应到元素的特性上。

移除特性
removeAttribute()

attributes属性
所有的特性都保存在这个属性当中。

ps：这个属性实质上是一个对象集合，它具有一些查找集合元素的方法。

ps：如果要遍历元素的特性，就可以使用这个属性。

创建元素
大部分浏览器支持：document.createElement()

ps：大部分浏览器的该方法传递的参数为标签的名称，例如div、a等；

document.createElement('div');

但是IE浏览器还支持传递整个标签，例如:

document.createElement("<div></div>");

ps：ie7即其以下的浏览器都存在着bug，解决方式是使用下面的方法：

if(client.browser.ie && client.browser.ie<=7){
    //低版本无法设置动态创建的name特性
    var iframe=document.createElement("<iframe name=\"myframe\"></iframe>");

    //低版本无法使用reset()方法重置input元素
    var input=document.createElement("<input type=\"checkbox\">");

    //低版本创建的reset按钮无法重置表单
    var button=document.createElement("<button type=\"reset\"></button>");

    //低版本创建的单选按钮，即使name相同，但是也无法实现单选
    var radio1=document.createElement("<input type=\"radio\" name=\"choice\" value=\"1\">");
    var radio1=document.createElement("<input type=\"radio\" name=\"choice\" value=\"2\">");
}

元素子节点
通过childNodes属性获取直接子节点。
ie浏览器在识别子节点时不会计算文本节点（空白符），而在其他浏览器会记录文本节点。

<ul>
    <li>1</li>
    <li>2</li>
    <li>3</li>
</ul>
<!--ul子节点数：ie浏览器返回3，其他浏览器返回7-->

<ul><li>1</li><li>2</li><li>3</li></ul>
<!--所有浏览器都返回3-->

浏览器-浏览器缓存机制

304状态和200状态

当客户端的浏览器向服务器第一次发送HTTP请求时，服务器在给客户端响应时，在响应上提供一些标记（last-modified或者etag），因为是第一次响应，返回的必然是200状态。

缓存过期时，当客户端第二次向服务器发送HTTP请求，该请求头中会有这些标记，服务器会利用这些标记进行比对，如果文件没有被修改，则返回304状态，告诉浏览器上一次的文件还可以使用（它们是留存在cache中的）；如果被修改了，则返回最新的文件信息，返回的是200状态。

两种缓存

协商缓存：会和服务器商量，看看文件是否修改过，缓存的内容时时保持在最新状态。
彻底缓存：在到期之间都不会和服务器交互。即使文件内容发生改变，加载网页还在使用缓存中旧的内容。

last-modified和etag都属于协商缓存，expires和cache-control属于彻底缓存。

详解两种标记

last-modified
这个标记在服务器响应客户端时提供，格式是这样的：

last-modifeid：时间（精确到秒）

如果缓存到期，客户端第二次请求相同的url时，请求头便会带上这个时间信息，但是格式会发生一些变化：

if-modified-since：时间点（和上面的时间是一样的）

服务器收到这个事件信息，如果在这个事件之后，文件有更新，那么发送新的文件，返回200状态，否则返回304状态。

etag
区别于last-modified标志，它的格式是这样的：

Etag:“5d8c72a5edda8d6a:3239″

如果缓存到期，客户端第二次请求相同的url时，请求便会带上这个信息，但是格式会发生一些变化：

If-None-Match:“5d8c72a5edda8d6a:3239″

服务器进行比对，如果不一致，则发送新的文件，返回200状态，否则返回304状态。

为什么会出现etag？
1.Last-Modified标注的最后修改只能精确到秒级，如果某些文件在1秒钟以内，被修改多次的话，它将不能准确标注文件的修改时间
2.如果某些文件会被定期生成，当有时内容并没有任何变化，但Last-Modified却改变了，导致文件没法使用缓存
3.有可能存在服务器没有准确获取文件修改时间，或者与代理服务器时间不一致等情形
（from 百度上某大牛的总结）

ps:两种标记可以一起使用

js-概念小题(1)

第一题

if (!("a" in window)) {
        var a = 1;
}
alert (a);

输出结果：underfined
分析原因：因为预编译的原因，a变量的定义会比if语句先发生，但不会进行赋值操作，所以是underfined

这道题，我还不晓得预编译，蒙也能蒙对。

第二题

var a = 1,
        b = function a (x) {
            x && a (--x);
        };
alert (a);

输出结果：1
分析原因：这样创建函数的方式并不是函数声明方式，a函数并不是全局的。

第三题

function a (x) {
        return x * 2;
}
var a;
alert (a);

输出结果：function….
分析原因：后定义的变量a并没有进行赋值，所以a还是原来的函数，并不会覆盖，所以…..

第四题

function b (x, y, a) {
        arguments[2] = 10;
        alert (a);
}
b(1, 2, 3);

输出结果：10
分析原因：参数会和方法的arguments属性保持同步，一个变，另外一个也会跟着变。

第五题

function a () {
        alert (this);
}
a.call (null);

输出结果：window
分析原因：call()第一个参数为函数执行环境，如果参数是null，会和不传递任何参数有同样的结果，函数的执行环境都是全局的。

这些题在实际开发中并没有什么卵用，放出来只是帮助你深刻了解js的一些概念而已。

js-客户端检测

客户端检测

客户端检测分为三种检测，分别是能力检测，怪癖检测以及用户代理检测。

ps：优先使用能力检测，其次考虑怪癖检测，最后再是用户代理检测。

ps：客户端检测是一种补救措施，不到万不得已，不要使用客户端检测。尽量找到最通用的方案，也就是说各大浏览器都可以识别并正常运行的代码。

能力检测

能力检测不是在识别浏览器，而是识别浏览器的能力，具体的指某个方法支不支持或者某个属性具不具备。
两个重要的概念：

1.先检测达成目的最常用的特性

function getElement(id){
    if(document.getElementById){
        return document.getElementById(id);
    }else if(document.all){
        return document.all[id];
    }else{
        throw new Error("没有这样的方法");
    }
}//IE5之前不支持document.getElementById，但是IE6+、chrome、safari等其他浏览器都支持，所以它被认为是最常用的特性。

2.测试实际要用到的特性。就是说，你要用哪个方法，就测那个方法存不存在。存在的话提供一套方案，不存在的话提供另外一套方案。

function getWindowWidth(){
    if(document.all){
        return document.documentElement.clientWidth;//错误的用法
    }else{
        return window.innerWidth;
    }
}//document.all存在并不意味着clientWidth也存在

能力检测并不是浏览器检测
ps：能力检测并不能确定浏览器。

ps：当需要使用某些特定的浏览器特性时，最好时一次性检测所有的特性，而不要分开检测。

怪癖检测

怪癖检测时想要检测浏览器存在哪些bug。

ps：最好在一开始就执行此类检测。

例子：

var hasEnumShadowsQuick(){
    var o={toString:function(){}};
    var count=o;
    for(var p in o){
        if(p == "toString"){
            count++;
        }
    }
    return (count > 1);
}();

如果是在safari3以前的版本下运行会返回true，因为他会枚举被隐藏的属性。

ps:另外一种检测怪癖的方式是通过try-catch。

用户代理检测

用户代理字符串 

var ua=navigator.userAgent;//获取用户代理字符串

用户代理字符串提供的信息：平台、呈现引擎、版本号（浏览器、内核、Mozilla）等。

浏览器及内核

IE ———— Trident内核
FF ———— Gecko内核
Safari、Chrome ———– Webkit内核
Konqueror ———– KTHML内核
Opera ———– Gecko内核

用户代理字符串技术

识别呈现引擎–>识别浏览器–>识别平台–>识别Windows操作系统–>识别移动设备–>识别游戏系统

var client=function(){

    var engine={
        //内核
        ie: 0,
        gecko: 0,
        webkit: 0,
        khtml:0,
        operaL0,

        ver: null
    };
    var browser={
        //浏览器
        ie: 0,
        firefox: 0,
        safari: 0,
        konq: 0,
        opera: 0,
        chrome 0,

        ver: null
    }
    var system={
        //平台
        win: false,
        mac: false,
        xll: false

        //移动设备
        iphone: false,
        ipod: false,
        ipad: false,
        ios: false,
        android: false,
        nokiaN: false,
        winMobile: false,

        //游戏系统
        wii: false,
        ps: false 
    }


        //以下是检测的代码

        //检测引擎和浏览器
        //通过正则匹配找到用户代理字符串中的版本号
        var ua=navigator.userAgent;//获取代理字符串

        if(window.opera){//opera最会伪装自己
            engine.ver=browser.ver=window.opera.version();
            engine.opera=browser.ver=parseFloat(engine.ver);
        }else if(/AppleWebkit\/(\S+)/.test(ua)){//webkit的用户代理字符串中有KHTML和Gecko字样
            engine.ver=RegExp["$1"];
            engine.webkit=parseFloat(engine.ver);

            //确定是chrome？safari？
            if(/Chrome\/(\S+)/.test(ua)){//chrome
                browser.ver=RegExp["$1"];
                browser.chrome=parseFloat(browser.ver);
            }else if(/Version\/(\S+)/.test(ua)){//safari
                browser.ver=RegExp["$1"];
                browser.safari=parseFloat(browser.ver);
            }else{//近似估计
                var safariVersion=1;

                if(engine.webkit<100){
                    safariVersion=1;
                }else if(engine.webkit<312){
                    safariVersion=1.2;
                }else if(engine.webkit<412){
                    safariVersion=1.3;
                }else{
                    safariVersion=2;
                }
                browser.safari=browser.ver=safariVersion;
            }
        }else if(/KHTML\/(\S+)/.test(ua)){//KHTML的用户代理字符串中有Gecko字样
            engine.ver=browser.ver=RegExp["$1"];
            engine.khtml=browser.konq=parseFloat(engine.ver);
        }else if(/rv:([^\)]+)\)Gecko\/\d{8}/.test(ua)){
            engine.ver=RegExp["$1"];
            engine.gecko=parseFloat(engine.ver);

            if(/Firefox\/(\S+)/.test(ua)){
                browser.ver=RegExp["$1"];
                browser.firefox=parseFloat(browser.ver);
            }
        }else if(/MISE([^;]+)/.test(ua)){
            engine.ver=browser.ver=RegExp["$1"];
            engine.ie=browser.ie=parseFloat(engine.ver);
        }

        browser.ie=engine.ie;
        browser.opera=engine.opera;

        //检测平台
        var p=navigator.platform;
        system.win=p.indexOf("Win")==0;
        system.mac=p.indexOf("Mac")==0;
        system.xll=(p=="xll") || (p.indexOf("Linux")==0);

        //检测windows操作系统
        if(system.win){
            if(/Win(?:dows)?([^do]{2})\s?(\d+\.\d+)?/.test(ua)){
                if(RegExp["$1"]=="NT"){
                    switch(RegExp["$2"]){
                        case "5.0":
                            system.win="2000";
                            break;
                        case "5.1":
                            system.win="xp";
                            break;
                        case "6.0":
                            system.win="Vista";
                            break;
                        case "6.1":
                            system.win="7";
                            break;
                        default:
                            system.win="NT";
                            break;
                    }
                }else if(RegExp["$1"]=="9x"){
                    system.win="ME";
                }else{
                    system.win=RegExp["$1"];
                }
            }
        }

        //移动设备
        system.iphone=ua.indexOf("iPhone")>-1;
        system.ipod=ua.indexOf("iPod")>-1;
        system.ipad=ua.indexOf("iPad")>-1;
        system.nokiaN=ua.indexOf("NokiaN")>-1;

        //window mobile
        if(system.win == "CE"){
            system.winMobile=system.win;
        }else if(systen.win=="Ph"){
            if(/Windows Phone OS (\d+.\d)/.test(ua)){
                system.win="Phone";
                systen.winMobile=parseFloat(RegExp["$1"]);
            }
        }

        //检测IOS版本
        if(system.mac && ua.indexOf("Mobile")>-1){
            if(/CPU(?:iPhone)?OS(\d+_\d+)/.test(ua)){
                system.ios=parseFloat(RegExp.$1.replace("_","."));
            }else{
                system.ios=2;//只能猜测
            }
        }

        //检测Android版本
        if(/Android(\d+\.\d+)/.test(ua)){
            system.android=parseFloat(RegExp.$1);
        }

        //游戏系统
        system.wii=ua.indexOf("Wii")>-1;
        system.ps=/playstation/i.test(ua);


        return {
            engine: engine,
            browser: browser,
            system: system
        };    
}();

ps:不同的浏览器具备不同的能力，同一款浏览器在不同的平台具备不同的能力；不能直接准确地使用能力检测或怪癖检测；为了跟踪分析等目的需要知道确切的浏览器。
