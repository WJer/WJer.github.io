js-闭包

闭包

闭包就是指有权访问另一个函数作用域中的变量的函数。

执行环境和作用域

执行环境定义了变量或函数有权访问的其他数据。

每个执行环境都有一个对应的变量对象，eg：window对象，环境内声明的变量和方法都保存在这个对象中。

函数执行时都会创建自己的执行环境及相应的作用域链。

作用域链本质上是一个变量对象的指针列表。函数在访问某一个变量时就会从作用域链一层一层的寻找。

var name="china";
function provice(){
    var name="anhui";
    function city(){
        var name="hefei";
    }
}

分析下这个简单的例子：
1.三个执行环境：全局执行环境，provice()执行环境，city()执行环境
2.city()函数执行时会创建自己的作用域链，依次是city()的活动对象，provice()的活动对象，window对象。
3.provice()函数执行时会创建自己的作用域链，依次是provice()的活动对象，window对象。
4.查找变量时就是按照从最里面到最外面的作用域链访问变量。
5.当city()执行完时，这个函数的执行环境就会退出环境栈，并且对应的变量对象也会相应的销毁。同理 provice()也是。

闭包

function a(){
    var name=1;
    return function (){
        var age=2;
    }
}
var b=a();//b的作用域链包括a函数的活动对象
b=null;//匿名函数不再有引用，垃圾回收机制会安全销毁b函数的作用域链。

以上看来，在a函数的外部可以访问a内部声明的变量和函数。

闭包的问题

问题例子：

function createF(){
    var result=new Array();
    for(var i=0;i<=5;i++){
        result[i]=function(){
            return i;
        }
    }
}
var fs=createF();
for(var i=0;i<=5;i++){
    console.log(fs[i]());//输出5个5
}

本来每个函数都输出自己的i，结果全都输出相同的数。
原因：函数数组中每个函数的作用域链都保存着createF()函数的活动对象，所以它们访问的是同一个i。

解决方式：

function createF(){
    var result=new Array();
    for(var i=0;i<=5;i++){
        result[i]=function(num){
            return function(){//又创建了一个闭包
                return num;//每个函数都有自己的num变量
            }
        }(i);
    }
    return result；
}

js-继承

继承

javascript是通过原型链来实现继承。
实现继承的方式有很多，但总的思路就是原型链，本人总结了一下，如下：

原型链

function Super(){
    ......
}
Super.prototype.saySupername=function(){
    ......
}//此处代表对超类型的原型的操作
function Sub(){
    ......
}
Sub.prototype=new Super();

通过上面的方式，便实现了继承。

注意：
1.这种方式的原型链在最顶层还有一个Object.prototype，不要忽略。
2.可以通过instanceof和isPrototypeOf来确定实例和原型的关系
3.存在的问题：原型中的值会被共享、在创建子类型的实例时无法向超类型的构造函数传递参数。

借用构造函数

function Super(){
    ......
}
function Sub(){
    ......
    Super.call(this,[参数]);//超类型的构造函数当普通函数使用
}

注意：
1.这种方式可以在创建子类型的实例时给超类型的构造函数传递参数。
2.存在的问题：当只借用构造函数这种方式时，那么就无法做到函数复用，除非使用原型。

组合继承

function Super(name){
    this.name=name;
}
Super.prototype.sayName=function(){
    return this.name;
}
function Sub(name,age){
    Super.call(this,name);
    this.age=age;
}
Sub.prototype=new Super();

就是把原型链和借用构造函数两种方式组合起来使用。
注意：1.这是最常用到的继承模式。

原型式继承

function object(o){
    function F(){};
    F.prototype=o;
    return new F();
}

注意：
1.使用Object.create(o);来代替上面的函数，这是ECMAScript5规范了的方法。
2.使用条件：只想使两个对象保持类似的问题。
3.存在的问题：包含引用类型的属性都会共享相应的值，也就是无法有私有的属性和方法。

寄生式继承

function createAnother(o){
    var clone=object(o);
    clone.sayHi=function(){//这里可以创建一些私有的属性和方法
        ......
    }
    return clone;
}

注意：
1.和构造函数一样，若只是使用这种方式，那么就无法达到函数复用。

寄生组合式继承

先看看组合继承的缺点：重复执行相同的函数两次

function Super(name){
    this.name=name;
    this.color=[1,2,3];
}
Super.prototype.sayName=function(){
    console.log(this.name);
}
function Sub(name,age){
    Super.call(this,name);//执行第一次
    this.age=age;
}
Sub.prototype=new Super();//执行第二次

改变后：

function inheritPrototype(Sub,Super){
    var prototype=object(Super.prototype);
    prototype.constructor=Sub;
    Sub.prototype=prototype;
}
function Super(name){
    this.name=name;
    this.color=[1,2,3];
}
Super.prototype.sayName=function(){
    console.log(this.name);
}
function Sub(name,age){
    Super.call(this,name);//执行第一次
    this.age=age;
}
inheritPrototype(Sub,Sup);

注意：
普遍认为这是最理想的继承方法。

nodejs-supervisor

使用supervisor提高调试效率

每次改完项目的js文件，都要重新node一下，实在是麻烦。

后来知道了supervisor，一个强大而又实用的工具，很适合调试代码。

如何安装

通过

supervisor app.js

启动。当你修改项目之后，它会检测到你的修改，然后自动重启项目，不需要你再输入命令来启动项目。

直接用npm安装既可，键入命令: 

npm -g install supervisor

自己电脑的问题

不知道为什么，我每次安装都不是全局的，安装成功仍然无法使用，不过没关系，手动去实现。例如install到nodejs的文件夹下,以防出错，你还是去nodejs/bin下查看是否有supervisor和node-supervisor这个文件。

sudo ln -s  nodejs/bin/supervisor /usr/local/bin/supervisor

通过软连接的方式来实现全局。

hexo-使用'hexo d'报错

“ERROR Deployer not found” 错误（for linux）

hexo 升级到3.0之后，再使用

hexo deploy

会出现“ERROR Deployer not found”的错误，看了很多别人的评论，才知道原因。

原因是缺少”hexo deploy”的相关模块。

既然缺少相关模块，那我们安装吧。但是安装也需要方法的，先给出安装步骤：

1.进入博客所在的文件夹里

没错，这步是必须的，每个项目都有一个node_modules文件夹，正是里面缺少”hexo deploy”的相关模块,所以导致无法使用hexo deploy。网上都没有说这一步，害的菜鸟的我在坑里滚了很长时间，最后才发现这一步的关键。

2.安装模块

在终端输入

npm install hexo-deployer-git  --save
3.修改type

博客项目都有个“_config.yml”的文件，修改里面的type为“git”(以前是github；还要注意冒号后面空一格)

好了，可以使用hexo deploy了，搞定！！

如果还报了其他的错误，考虑有没有设置git config。当初我是刚安装的git，就忘了设置config global ×××，结果蛋疼了很久。

nodejs-使用npm安装模块失败

npm install express，报错！！！
npm install formidable，报错！！！
我的天！这什么情况啊！！！
起初以为是自己改设置改的凌乱了，就决定重新安装nodejs，结果还是install不了，那可咋整？
百度一下，分析了一下原因，主要还是受到墙的限制，无法正常访问模块所在的服务器，需要修改代理什么的，也就是换个服务器去下载。
百度了许多，现在总结一下解决办法：

办法一：

百度上有很多代理，我重点推荐淘宝的镜像，戳进去看看吧，上面有教程。
其实这个方法很简单，就是安装了一个cnpm取代npm，通过

cnpm install 模块名字

来下载安装模块，很简单吧。

办法二：

设置其他的代理，打开终端（或者cmd），输入

npm config set registry = http://registry.cnpmjs.org

我们再输入

npm config ls

可以看到我们刚才设置的那一项。
然后通过

npm install 模块名字

显示出express下载成功。
我们运行下项目，奇怪！明明显示已经install成功了，在node_modules中也能找到相应的文件夹，怎么还是 not found module。不要怕，往后看。如果可以运行项目的话，那就此止步吧。
我们进入nodejs文件目录，在node_modules文件里找到所安装模块的文件夹（例如：名为“formidable”的文件夹），我们把它复制。
接着进入项目的根目录里，创建一个node_modules文件夹，把刚刚复制的文件夹保存到这个node_modules文件夹里面。好了node index.js ,我们发现项目可以运行了。bingo！

注意：我们可以通过

npm config delete 键名

来删除新添加的配置项。

这个必须安装，我之前没有安装，后面的步骤一直报错，苦恼了好几天。

3.拷贝如下代码到文件sublime_imfix.c文件中，该文件需要自己创建，随便放到那里都行

进入保留该文件的目录，在终端输入

4.中文输入

我们试试能不能用吧。
在终端中输入如下命令：

LD_PRELOAD=./libsublime-imfix.so subl             #subl是安装好SublimeText 3后的程序启动命令

如果一切正常，在启动之后，搜狗输入法就能可以使用了。

5.最后的设置

将libsublime-imfix.so拷贝到系统库的默认路径中：

将

修改为

js-递归

递归

函数的递归很简单，但是在js中要注意一下递归的问题。

function jiecheng(num){
    if(num<=1){
        return 1;
    }else{
        return num * jiecheng(num - 1);
    }
}

这是个简单的阶乘，但是存在个问题。

var a=jiecheng;
jiecheng=null;
a(4);//报错

因为函数的内部是通过调用函数名来实现递归，一旦函数名为null值了，所以就导致错误。

解决办法：

function jiecheng(num){
    if(num<=1){
        return 1;
    }else{
        return num * arguments.callee(num - 1);
    }
}

这种方式是通过使用arguments.callee来代替函数名，但是又会有新的问题：严格模式下，脚本无法访问arguments.callee。

解决办法：

var jiecheng=(function f(num){
    if(num<=1){
        return 1;
    }else{
        return num*f(num-1);
    }
});

不管怎么变，f函数名都是有效的，而且也能在严格模式下使用。
