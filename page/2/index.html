js-事件-事件流

事件流

如今浏览器的事件流分为两种：事件冒泡和事件捕获。

为什么会出现事件流呢？
假如子div和父div都添加了click事件，当我们点击子div时，浏览器应该先处理子div的事件还是父div的事件呢？为了解决这个问题，因此产生了事件流。

事件冒泡：先执行子div,再是父div
事件捕获：先执行父div,再是子div
两者的顺序刚好相反。

浏览器支持

IE8只支持事件冒泡，IE9和其他浏览器两者都支持。

如何切换事件流

如果我们想指定使用哪种事件流，可以使用下列的方法来添加事件。

addEventListener(交互方式,处理函数,true/false);
//true为事件捕获，false为事件冒泡

js-DOM扩展-元素遍历

元素遍历

为什么要新增加元素遍历？
有些浏览器在获得某个DOM节点的childNodes时会将空文本节点包括进去，为了弥补这样的差异，Element Traversal新定义了一组属性。

  • childElementCount:返回子元素的个数
  • firstElementChild:返回第一个子元素
  • lastElementChild:返回最后一个子元素
  • previousElementSibling:返回某个元素前一个同辈子元素
  • nextElementSibling:返回某个元素后一个同辈子元素

支持：IE9+,FF3.5+,SF4+,CH,OP10+

选择符API

选择符API level1的核心

通过选择器来获得DOM对象

核心：querySelector()和querySelectorAll()

支持的浏览器：IE8+,ff3.5+,SF3.1+,CH,OP10+

可以通过document或者元素节点来调用这两种方法。

querySelector()
返回选择器匹配到的第一个元素节点,没有匹配到则返回null

<div id="parent">

</script>

querySelectorAll()
返回选择器匹配到的所有节点，该方法返回一个NodeLIst的实例。没有匹配到，NodeList就是空的。

ps：NodeLIst底层实现不是对文档的动态查询，而是一组元素的快照。？？？？

选择符API level２的核心

matchesSelector()
如果调用的元素与选择器匹配，则返回true;否则返回false。

其他浏览器可能有相同功能的方法，但是名字不同。

下面是实现此功能并兼容各浏览器的包装函数：

function matchesSelector(element,selector){
    if(element.matchesSelector(selector)){
        return element.matchesSelector(selector);
    }else if(element.msMatchesSelector(selector)){//兼容ie9+
        return element.msMatchesSelector(selector);
    }else if(element.mozMatchesSelector(selector)){//兼容ff3.6+
        return element.mozMatchesSelector(selector);
    }else if(element.webkitMatchesSelector(selector)){//兼容sf和ch
        return element.webkitMatchesSelector(selector)
    }else{
        throw new Error("not suppoted");
    }
}

js-DOM节点类型-Attr类型

Attr类型

比较冷门的类型，不经常用到。
特征
nodeType:2;
nodeName:特性的名称;
nodeValue: 特性的值;
parentNode: null;

ps：通常操作特性时通过getAttribute()、setAttribute()、removeAttribute()等方法，很少使用特性节点。

属性
特性节点的属性：name、value、specified。

ps：因为ie7及以下版本的浏览器存在着问题，一个元素节点的attributes属性可能会返回几十个特性，这时需要specified属性来进行筛选。

方法
获取特性节点：

element.getAttributeNode(attrName);

or

element.attributes[i];

当然你也可以通过特性节点来获取特性的值，只是不推荐使用。

添加特性节点：

element.setAttributeNode(attrNode);

or

element.setAttribute(name,value);

bootstrap-响应式启示

启示一

使用百分比

像栅格系统那样，通过设置宽度为百分比来适应浏览器的屏幕。

缺点：如果某行的元素数为5，那百分比应设置为20%；如果一行的元素为9，那就得人工计算；如果数目再次改变，那么又得在css样式中修改这个数值。

虽然得计算，但确实是个很好的办法。

启示二

这个思路不需要去计算除法。

我们在写导航栏的时候用的都是ul li等元素，那么以此为例：

ul>li{
    display:table-cell;
    width:1%; 
}

为什么是1%？
至今没想出来。

这样一设置，导航栏各li元素的宽度就会随着屏幕的宽度变化而变化，不论你怎么改变，它们都会平均的分配宽度。

关于table-cell值：
这个属性不能和float、position:absolute同时使用，它会被这些属性破坏。
IE8+浏览器及其他浏览器都支持。

启示三

学会使用媒体查询，根据不同的浏览器宽度使用不同的样式。

@media(最小宽度或最大宽度){
    ......
}

栅格系统可以说是bootstrap的核心之一，响应式设计就是建立在这个基础之上。

实现原理

定义容器，调整内外边距，平分12份，媒体查询

这四个合起来就能实现栅格系统。
ps：12等分是默认的，我们可以重新编译LESS来修改这个数值。

一个简单的例子：

<div class="container">

响应式栅格

超小型屏幕：

  • <768px
  • 任何分辨率下都水平排列
  • 最大容器宽度时自动
  • .col-xs-
  • 可嵌套，可排序，可偏移

小型屏幕设备：

  • <992px &&="">=768px
  • 小于768时，垂直排列；超过768，水平排列
  • 最大容器宽度是750px
  • .col-sm-
  • 可嵌套，可排序，可偏移

中等屏幕设备：

  • <1200px &&="">=992px
  • 小于992，垂直排列；超过992，水平排列
  • 最大容器宽度是970px
  • .col-md-
  • 可嵌套，可排序，可偏移

大屏幕设备：

  • >1200px
  • 小于1200时，垂直排列；超过1200，水平排列
  • 最大容器宽度是1170px
  • .col-lg-
  • 可嵌套，可排序，可偏移

ps：向大兼容，小于媒体查询中的最小宽度，就不起作用了。
