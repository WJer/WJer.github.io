js-模拟实现java类

##模拟实现java类##
要想模拟出java类的所有结构，那当然不可能了。js是弱类型语言，java是强类型语言。
可以模拟的部分：

  • 静态方法和静态属性
  • 实例方法和实例属性
  • 继承
  • 无法被修改的属性或者方法
  • 私有属性和方法

模拟狗

定义一个狗的类，通过构造函数来模拟。同时，构造函数内部的属性和方法模拟实例属性和实例方法。

function Dog(name){
    this.name=name||"";
    this.say=function(){
        console.log(this.name);
    }
}

狗继承哺乳动物，通过原型实现。

Dog.prototype.class="哺乳动物";
Dog.prototype.behavior=function(){
    console.log("具有哺乳行为。");
}

构造函数事实上是一个对象，通过对这个对象添加属性和方法来模拟静态方法和静态属性。

Dog.LEGS=4;//有４条腿
Dog.shout=function(){
    console.log("旺旺");
}

js-事件-提高性能

性能问题

当我们创建事件处理程序时，首先需要找到DOM对象，然后再绑定什么事件如何处理。

如果一个页面的事件很多的话，首先访问的DOM对象就会很多，其次每个事件都是一个对象，那么内存中的对象也就越多，这样一来，性能就会下降。

如何从事件的角度提高性能

事件委托
例如：

<ul>
    <li>1</li>
    <li>2</li>
    <li>3</li>
    <li>4</li>
</ul>

如果要为这个列表的li添加点击事件，传统的方法是分别获取这四个li，然后每个li添加一个点击事件，这样内存中就会出现四个相同的点击事件，内存真的好浪费呀，代码还长。

而事件委托是这样的：

var ul=document.getElementByTagName('ul')[0];
ul.onclick=function(e){
    var li=e.target;
    switch(li.id){
        .......//针对不同的li执行不同的动作
    }
}

可以看出，事件委托就是把事件绑定到高层元素，然后通过target来判断具体是哪个子元素发生事件。只需一个事件就能达到多种事件的效果。

ps:可以看出事件必须冒泡，才有可能事件委托。

ps:可以使用事件委托的事件：click,mousedown,mouseup,keydown,keyup,keypress

ps:mouseout和mouseover无法使用事件委托。因为鼠标从父元素到子元素和从子元素到父元素都会触发事件，这样就无法确定是不是子元素触发的事件。

移除事件处理程序
设想一下这样的情况，假如我们通过js删除了某个元素节点，但是这个节点已经绑定了某个事件，虽然删除了该节点，但是事件仍然留在内存当中，不能被垃圾回收。甚至对DOM节点的引用也保留在内存里。

移除事件处理程序就是如果我们要修改文档结构，例如删除某个节点，我们事先也知道这个节点上有事件监听，我们应该先删除事件处理程序，再删除节点。垃圾回收机制会将事件处理程序回收，从而清理内存。

早起的IE浏览器将页面卸载后还会保留绑定在该页面上的所有事件，这样每关闭一个页面，内存的容量就要减少许多。针对这样的情况，最好的做法就是通过unload事件来移除手游的事件处理程序。

ps:事件委托也可以避免这样的情况，因为我们把事件绑定在高层元素，子元素删了有何妨？什么都会影响嘛。

js-事件-事件流

事件流

如今浏览器的事件流分为两种：事件冒泡和事件捕获。

为什么会出现事件流呢？
假如子div和父div都添加了click事件，当我们点击子div时，浏览器应该先处理子div的事件还是父div的事件呢？为了解决这个问题，因此产生了事件流。

事件冒泡：先执行子div,再是父div
事件捕获：先执行父div,再是子div
两者的顺序刚好相反。

浏览器支持

IE8只支持事件冒泡，IE9和其他浏览器两者都支持。

如何切换事件流

如果我们想指定使用哪种事件流，可以使用下列的方法来添加事件。

addEventListener(交互方式,处理函数,true/false);
//true为事件捕获，false为事件冒泡

js-DOM扩展-元素遍历

元素遍历

为什么要新增加元素遍历？
有些浏览器在获得某个DOM节点的childNodes时会将空文本节点包括进去，为了弥补这样的差异，Element Traversal新定义了一组属性。

  • childElementCount:返回子元素的个数
  • firstElementChild:返回第一个子元素
  • lastElementChild:返回最后一个子元素
  • previousElementSibling:返回某个元素前一个同辈子元素
  • nextElementSibling:返回某个元素后一个同辈子元素

支持：IE9+,FF3.5+,SF4+,CH,OP10+

js-DOM扩展-选择符API

选择符API

选择符API level1的核心

通过选择器来获得DOM对象

核心：querySelector()和querySelectorAll()

支持的浏览器：IE8+,ff3.5+,SF3.1+,CH,OP10+

可以通过document或者元素节点来调用这两种方法。

querySelector()
返回选择器匹配到的第一个元素节点,没有匹配到则返回null

<div id="parent">
    <div class="sub">1</div>
    <div class="sub">2</div>
    <div class="sub">3</div>
    <div class="sub">4</div>
</div>
<script>
    var div=document.getElementById('parent');
    var sub1=div.querySelector('.sub');//返回第一子节点
</script>

querySelectorAll()
返回选择器匹配到的所有节点，该方法返回一个NodeLIst的实例。没有匹配到，NodeList就是空的。

ps：NodeLIst底层实现不是对文档的动态查询，而是一组元素的快照。？？？？

选择符API level２的核心

matchesSelector()
如果调用的元素与选择器匹配，则返回true;否则返回false。

其他浏览器可能有相同功能的方法，但是名字不同。

下面是实现此功能并兼容各浏览器的包装函数：

function matchesSelector(element,selector){
    if(element.matchesSelector(selector)){
        return element.matchesSelector(selector);
    }else if(element.msMatchesSelector(selector)){//兼容ie9+
        return element.msMatchesSelector(selector);
    }else if(element.mozMatchesSelector(selector)){//兼容ff3.6+
        return element.mozMatchesSelector(selector);
    }else if(element.webkitMatchesSelector(selector)){//兼容sf和ch
        return element.webkitMatchesSelector(selector)
    }else{
        throw new Error("not suppoted");
    }
}
