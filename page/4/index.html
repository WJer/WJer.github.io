js-DOM节点类型-Attr类型

Attr类型

比较冷门的类型，不经常用到。
特征
nodeType:2;
nodeName:特性的名称;
nodeValue: 特性的值;
parentNode: null;

ps：通常操作特性时通过getAttribute()、setAttribute()、removeAttribute()等方法，很少使用特性节点。

属性
特性节点的属性：name、value、specified。

ps：因为ie7及以下版本的浏览器存在着问题，一个元素节点的attributes属性可能会返回几十个特性，这时需要specified属性来进行筛选。

方法
获取特性节点：

element.getAttributeNode(attrName);

or

element.attributes[i];

当然你也可以通过特性节点来获取特性的值，只是不推荐使用。

添加特性节点：

element.setAttributeNode(attrNode);

or

element.setAttribute(name,value);

js-DOM节点类型-Comment类型

Comment类型

特征
nodeType:8;
nodeName:#comment;

parentNode: Document or Element;

ps：该节点无法访问到html后面的注释，也就是最最外层的注释。

创建注释节点

document.createComment(text);//返回注释节点

js-DOM节点类型-Text类型

Text类型

特征
nodeType:3;
nodeName:#text;
nodeValue: 所包含的文本;
parentNode: element类型;

文本节点的属性和方法
1.通过nodeValue属性或者data属性获取文本内容。
2.

appendData(text);
deleteData(start,count);
insertData(start,text);
replaceData(start,count,text);
splitText(start);
substringData(start,count);

3.length属性获取文本的长度

获取文本节点

通过父节点的子节点属性（childNodes）获取
创建文本节点

document.createTextNode(text);//返回文本节点

合并文本节点

parentNode.normalize();

相邻的两个文本节点合并成一个节点。当然，浏览器在解析html时永远不会创建相邻的两个文本节点。

textNode.splitText(count);//返回新的文本节点，原文本节点删除分割的文本内容

bootstrap-响应式启示

启示一

使用百分比

像栅格系统那样，通过设置宽度为百分比来适应浏览器的屏幕。

缺点：如果某行的元素数为5，那百分比应设置为20%；如果一行的元素为9，那就得人工计算；如果数目再次改变，那么又得在css样式中修改这个数值。

虽然得计算，但确实是个很好的办法。

启示二

这个思路不需要去计算除法。

我们在写导航栏的时候用的都是ul li等元素，那么以此为例：

ul>li{
    display:table-cell;
    width:1%; 
}

为什么是1%？
至今没想出来。

这样一设置，导航栏各li元素的宽度就会随着屏幕的宽度变化而变化，不论你怎么改变，它们都会平均的分配宽度。

关于table-cell值：
这个属性不能和float、position:absolute同时使用，它会被这些属性破坏。
IE8+浏览器及其他浏览器都支持。

启示三

学会使用媒体查询，根据不同的浏览器宽度使用不同的样式。

@media(最小宽度或最大宽度){
    ......
}

title: bootstrap-栅格系统
date: 2015-12-12 16:36:34

category: bootstrap
栅格系统可以说是bootstrap的核心之一，响应式设计就是建立在这个基础之上。

实现原理

定义容器，调整内外边距，平分12份，媒体查询

这四个合起来就能实现栅格系统。
ps：12等分是默认的，我们可以重新编译LESS来修改这个数值。

一个简单的例子：

<div class="container">
    <div class="row">
        <div class="col-md-2">1</div>
        <div class="col-md-2">1</div>
        <div class="col-md-2">1</div>
        <div class="col-md-2">1</div>
        <div class="col-md-2">1</div>
        <div class="col-md-2">1</div>
    </div>
</div>

响应式栅格

超小型屏幕：

  • <768px
  • 任何分辨率下都水平排列
  • 最大容器宽度时自动
  • .col-xs-
  • 可嵌套，可排序，可偏移

小型屏幕设备：

  • <992px &&="">=768px
  • 小于768时，垂直排列；超过768，水平排列
  • 最大容器宽度是750px
  • .col-sm-
  • 可嵌套，可排序，可偏移

中等屏幕设备：

  • <1200px &&="">=992px
  • 小于992，垂直排列；超过992，水平排列
  • 最大容器宽度是970px
  • .col-md-
  • 可嵌套，可排序，可偏移

大屏幕设备：

  • >1200px
  • 小于1200时，垂直排列；超过1200，水平排列
  • 最大容器宽度是1170px
  • .col-lg-
  • 可嵌套，可排序，可偏移

ps：向大兼容，小于媒体查询中的最小宽度，就不起作用了。

跨设备组合：由于向大兼容的原因，小就无法兼容，那么可以再设置一套样式，如下：

<div class="container">
    <div class="row">
        <div class="col-sm-6 col-md-2">1</div>
        <div class="col-sm-6 col-md-2">1</div>
        <div class="col-sm-3 col-md-2">1</div>
        <div class="col-sm-3 col-md-2">1</div>
        <div class="col-sm-3 col-md-2">1</div>
        <div class="col-sm-3 col-md-2">1</div>
    </div>
</div>
<!--自己试试，效果必然不同-->

清除浮动：栅格里的所有格都有左浮动的样式，如果受到浮动影响了，可以使用.clearfix来清除浮动。使用如下

<div class="container">
    <div class="row">
        <div class="col-md-6">1内容好多好多好多好多,把高度撑大了</div>
        <div class="col-md-6">1</div>
        <!--此处开始受到浮动影响，新添加一个div-->

        <!----新添加的div----->
        <div class="clearfix"></div>
        <!----------------------------->

        <div class="col-md-6">1</div>
        <div class="col-md-6">1</div>
        <div class="col-md-6">1</div>
        <div class="col-md-6">1</div>
    </div>
</div>

基本使用方法

列偏移：

col-sm-offset-
col-md-offset-
col-lg-offset-

列嵌套

    <div class="row">

js-DOM节点类型-Element类型

Element类型

特征
nodeType:1;
nodeName:元素的标签名;
nodeValue: null;
parentNode: Document/Element;
ownerDocument: null;

特性
特性就是元素标签的属性，只是换了个名字而已。本人分析是为了区别元素标签的属性和DOM对象的属性而这样干的。

取得特性
两种方式：getAttribute()和直接访问DOM对象属性。

ps：并不是所有的特性都有对应的属性。对于一些公认的特性，才有与之对应的DOM对象的属性。

ps：在公认的特性中，有两类特殊的特性：style和onclick。两种方式取得的值不想同。由于这样的存在，开发人员在取得自定义特性使用getAttribute(),最普遍使用的还是DOM对象属性。

ps：ie浏览器中自定义特性也有对应的属性

设置特性
两种方式：setAttribute()和直接设置DOM属性。

ps：setAttribute()无法设置class和style特性。

ps：setAttribute()可以设置自定义特性，直接设置DOM属性只能设置属性，如果不是公共特性，并不能反应到元素的特性上。

移除特性
removeAttribute()

attributes属性
所有的特性都保存在这个属性当中。

ps：这个属性实质上是一个对象集合，它具有一些查找集合元素的方法。

ps：如果要遍历元素的特性，就可以使用这个属性。

创建元素
大部分浏览器支持：document.createElement()

ps：大部分浏览器的该方法传递的参数为标签的名称，例如div、a等；

document.createElement('div');

但是IE浏览器还支持传递整个标签，例如:

document.createElement("<div></div>");

ps：ie7即其以下的浏览器都存在着bug，解决方式是使用下面的方法：

if(client.browser.ie && client.browser.ie<=7){
    //低版本无法设置动态创建<iframe>的name特性
    var iframe=document.createElement("<iframe name=\"myframe\"></iframe>");

    //低版本无法使用reset()方法重置input元素
    var input=document.createElement("<input type=\"checkbox\">");

    //低版本创建的reset按钮无法重置表单
    var button=document.createElement("<button type=\"reset\"></button>");

    //低版本创建的单选按钮，即使name相同，但是也无法实现单选
    var radio1=document.createElement("<input type=\"radio\" name=\"choice\" value=\"1\">");
    var radio1=document.createElement("<input type=\"radio\" name=\"choice\" value=\"2\">");
}

元素子节点
通过childNodes属性获取直接子节点(所有子节点)。
ie浏览器在识别子节点时不会计算文本节点（空白符），而其他浏览器会记录文本节点(空白符)。

<ul>
    <li>1</li>
    <li>2</li>
    <li>3</li>
</ul>
<!--ul子节点数：ie浏览器返回3，其他浏览器返回7-->

<ul><li>1</li><li>2</li><li>3</li></ul>
<!--所有浏览器都返回3-->

添加子节点
appendChild(node);
两类特殊的元素标签
<script></script>和<style></style>

<script></script>
IE浏览器是不允许对script和style的DOM对象使用操作子节点的方法，例如。
appendChild()、removeChild()等。

动态加载js文件

加载外部js文件(各浏览器通用)：

function loadScript(){
    var script=document.createElement("script");
    script.type="text/javascript";
    script.src="url";
    document.body.appendChild(script);
}

加载行内js代码

function loadScriptString(){
    var script=document.createElement('script');
    script.type="text/javascript";
    try{//其它浏览器
        script.appendChild(document.createTextNode(code));
    }catch(ex){//ie浏览器
        script.text="code";
    }
    document.body.appendChild(script);
}

动态加载css

加载外部css文件

function loadCss(url){
    var style=document.createElement('style');
    style.rel="stylesheet";
    style.type="text/css";
    style.href=url;
    var head=document.getElementByTagName("head")[0];
    head.appendChild(style);
}

加载css样式

function loadCssString(css){
    var style=document.createElement("style");
    style.rel="stylesheet";
    style.type="text/css";
    try{//其他浏览器
        style.appendChild(document.createTextNode(css));
    }catch(ex){//IE浏览器
        style.stylesheet.cssText=css;
    }
    var head=document.getElementByTagName('head')[0];
    head.appendChild(style);
}

浏览器-浏览器缓存机制

304状态和200状态

当客户端的浏览器向服务器第一次发送HTTP请求时，服务器在给客户端响应时，在响应上提供一些标记（last-modified或者etag），因为是第一次响应，返回的必然是200状态。

缓存过期时，当客户端第二次向服务器发送HTTP请求，该请求头中会有这些标记，服务器会利用这些标记进行比对，如果文件没有被修改，则返回304状态，告诉浏览器上一次的文件还可以使用（它们是留存在cache中的）；如果被修改了，则返回最新的文件信息，返回的是200状态。

两种缓存

协商缓存：会和服务器商量，看看文件是否修改过，缓存的内容时时保持在最新状态。
彻底缓存：在到期之间都不会和服务器交互。即使文件内容发生改变，加载网页还在使用缓存中旧的内容。

last-modified和etag都属于协商缓存，expires和cache-control属于彻底缓存。

详解两种标记

last-modified
这个标记在服务器响应客户端时提供，格式是这样的：

last-modifeid：时间（精确到秒）

如果缓存到期，客户端第二次请求相同的url时，请求头便会带上这个时间信息，但是格式会发生一些变化：

if-modified-since：时间点（和上面的时间是一样的）

服务器收到这个事件信息，如果在这个事件之后，文件有更新，那么发送新的文件，返回200状态，否则返回304状态。

etag
区别于last-modified标志，它的格式是这样的：

Etag:“5d8c72a5edda8d6a:3239″

如果缓存到期，客户端第二次请求相同的url时，请求便会带上这个信息，但是格式会发生一些变化：

If-None-Match:“5d8c72a5edda8d6a:3239″

服务器进行比对，如果不一致，则发送新的文件，返回200状态，否则返回304状态。

为什么会出现etag？
1.Last-Modified标注的最后修改只能精确到秒级，如果某些文件在1秒钟以内，被修改多次的话，它将不能准确标注文件的修改时间
2.如果某些文件会被定期生成，当有时内容并没有任何变化，但Last-Modified却改变了，导致文件没法使用缓存
3.有可能存在服务器没有准确获取文件修改时间，或者与代理服务器时间不一致等情形
（from 百度上某大牛的总结）

ps:两种标记可以一起使用
